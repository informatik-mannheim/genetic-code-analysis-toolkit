The "Find" functionality can be used for searching for some patterns in a sequence.

Depending on the "Direction" switch the search is performed either from top to bottom, or from bottom to top, starting at the current position of the cursor in the sequence. The option "Wrap Search" can be used to begin the search from the start of the sequence provided that the end was reached without result. As soon as a match was found the sequence will be selected and highlighted in the editor.

By default the search input admits solely single nucleotide bases. For a more complex search, regular expressions can be used by enabling the "Regular Expression" checkbox. Any combination of the constructs listed in the following table can be used for the search for a given pattern.

Summary of regular-expression constructs

x[abc][^abc][a-zA-Z][a-d[m-p]][a-z&&[def]][a-z&&[^bc]][a-z&&[^m-p]].X?X*X+X{n}X{n,}X{n,m}XYX|Y(X)

a, b, or c (simple class)

d, e, or f (intersection)
a through z, except for b and c: [ad-z] (subtraction)
a through z, and not m through p: [a-lq-z] (subtraction)

To match e.g. all codons (tuples of length 3) ending in U, the regular expression ??U may be used. For tessera (tuples of length 4) starting with U and ending either in A or C the regular expression U??(A|C) applies.

For more information on regular expression constructs, please see the Java documentation on Patterns.
